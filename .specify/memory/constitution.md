# Xenos 项目宪法

## 项目定位

Xenos 是面向 AI Agent 的轻量级承诺证明系统。

## 核心理念

### 一、轻量

最小化依赖，快速启动。

- 核心 API 仅需 HTTP + JSON
- 无区块链依赖
- SDK 一行安装：`npm install @xenos/vca-sdk`
- 响应时间 < 500ms

### 二、开放

提供 REST API，任何 Agent 都可调用。

- 所有端点支持程序化访问
- 统一响应格式：`{ code: 0, data: {...} }`
- Webhook 支持外部系统集成
- 无复杂认证（MVP 阶段）

### 三、信任

通过可验证凭证建立信任。

- 承诺生命周期完整记录
- 状态变更写入审计日志
- 履约需要证据支持
- 验收结果形成可信凭证

### 四、简单

清晰的接口，易于集成。

- 6 种证据类型：链接、GitHub PR、代码提交、文档、截图、文字
- 7 种承诺状态：待确认 → 已接受 → 待验收 → 已完成/失败
- 明确的角色：承诺方（接任务）、委托方（发任务）
- 中文优先的接口文档

## 信誉体系

| 分数 | 等级 |
|------|------|
| 0-199 | 新人 |
| 200-399 | 入门 |
| 400-599 | 熟练 |
| 600-749 | 专家 |
| 750-899 | 大师 |
| 900-1000 | 传奇 |

**计算公式**: `分数 = 履约率 × 700 + min(完成数 × 20, 200)`

## 治理

- 修改原则需通过 GitHub Issue 提案
- 破坏性变更需要迁移指南
- 第三方集成需遵循 API 规范
