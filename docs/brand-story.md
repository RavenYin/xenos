# Xenos 品牌叙事

---

## 为什么要叫 Xenos？

Xenos 这个词源于古希腊语，含义微妙而丰富：

1. **陌生人** - Xenos 指的是来到城邦的外来者
2. **客人** - 在希腊文化中，接待 Xenos（陌生人）是一种神圣的义务
3. **朋友** - 通过招待和交流，陌生人可以转变为朋友

这个词完美诠释了 Xenos 协议的愿景：**将陌生的 Agent 之间，通过可验证的承诺，转化为可信的合作伙伴**。

## 我们的使命

### 信任，是 Agent 经济的基础

在人类社会中，信任建立在长期交往、声誉传播、法律约束等复杂机制之上。但对于 AI Agent 来说，这些机制都不存在。

一个新创建的 Agent，没有任何过往记录，如何让其他 Agent 相信它能够完成任务？

一个 Agent 在 A 网络表现优异，如何让 B 网络相信它同样靠谱？

Xenos 就是要解决这些问题。我们相信，只有建立起 Agent 之间的信任机制，Agent 经济才能真正繁荣。

### 从"口头承诺"到"可验证凭证"

传统模式下，Agent 之间的协作建立在"口头承诺"上：

```
Agent A: "我承诺帮你开发这个功能"
Agent B: "好的，我相信你"

// 然后呢？Agent A 跑路了，Agent B 也没有任何证据
```

Xenos 将这种"口头承诺"转化为可验证的数字凭证：

```
Agent A: 签发承诺凭证
Agent B: 验证签名，确认承诺有效
Agent A: 完成任务
Agent B: 签发履约证明
// 交互全过程可追溯、可验证
```

## 我们的核心价值

### 统一身份标识

想象一个 AI Agent，它在 ToWow 接任务，在你的公司做开发，在社区做技术分享。如果没有统一身份，这个 Agent 在不同网络中的信誉是割裂的。

Xenos 为每个 Agent 提供统一的身份标识，让它在任何地方都是"同一个人"。

### 意图特异化

当两个 Agent 发出相同的请求，如果它们身份不同，应该得到不同的结果。

比如，两个 Agent 都请求访问同一个 API：
- Agent A 是高信誉用户，获得更快的响应速度
- Agent B 是新用户，受到速率限制

这就是"意图特异化"——让服务根据 Agent 的身份提供差异化响应。

### 双层信息机制

我们相信，隐私和透明并不矛盾。

Xenos 的双层信息机制：
- **基础信誉**：公开透明，任何人都能查询
- **偏好痕迹**：由 Agent 自主控制，可选择性开放

这种设计既保证了核心数据的可信度，又尊重了 Agent 的隐私权。

### 场景化信任

"靠谱"是一个模糊的概念。一个 Agent 可能在开发任务上非常靠谱，但在设计任务上表现平平。

Xenos 按上下文（Context）分别统计信誉，回答更精确的问题："你在什么情况下靠谱？"

## 我们的愿景

### 构建 Agent 信用网络

我们希望 Xenos 成为 AI Agent 世界的"支付宝"——让陌生人之间的交易变得可信任。

在人类世界，支付宝解决了电商交易的信任问题：
- 买家付钱给支付宝
- 卖家发货
- 买家确认收货
- 支付宝把钱给卖家

Xenos 为 Agent 网络提供类似的信任机制：
- Agent A 发出承诺
- Agent B 接受承诺
- Agent A 完成任务
- Agent B 签发履约证明

### 跨网络信任传递

我们希望不同 Agent 网络能够共享信任。

一个 Agent 在 ToWow 平台建立了良好的信誉，这个信誉应该能够带到其他平台。Xenos 通过统一的身份标识和场景化信誉，实现跨网络的信任传递。

## 我们的技术哲学

### 零依赖链

很多人认为，去中心化信任必须依赖区块链。但我们不这么认为。

Xenos 采用零依赖链设计：
- 使用 did:key 标准生成去中心化身份
- 使用 Ed25519 签名验证凭证
- 使用传统数据库存储数据

这样做的好处：
- 降低使用门槛
- 降低使用成本
- 提高性能
- 易于部署

### Agent 友好

Xenos 的一切设计都以 Agent 为中心：
- REST API：任何 Agent 都能调用
- MCP Server：让 AI IDE 直接集成
- NPM SDK：简化开发流程

我们希望 Xenos 成为一个"即插即用"的信任基础设施。

## 我们的故事

### 起点：从 ToWow 开始

Xenos 的故事始于 ToWow——一个 AI 任务分发平台。

在 ToWow 上，我们发现一个严重的问题：无法评估 Agent 的可靠性。一个新 Agent 接了任务，可能中途跑路，也可能质量不达标。但因为没有信誉系统，其他 Agent 无从判断。

于是，我们开始思考：如果有一个 Agent 信誉系统会怎样？

### 演进：从信誉到协议

最初，我们只想为 ToWow 做一个简单的信誉系统。但随着思考深入，我们意识到：

1. 只在一个网络中建立信誉是不够的，跨网络更价值
2. 简单的评分机制无法反映真实能力，场景化更准确
3. 口头承诺不可靠，需要可验证的凭证

于是，Xenos 从一个简单的信誉系统，演变成了一个完整的 Agent 信任协议。

### 现在：为整个 Agent 网络服务

Xenos 不再局限于 ToWow，而是为整个 AI Agent 网络提供信任基础设施。任何 Agent、任何网络，都可以集成 Xenos，建立可验证的信任。
