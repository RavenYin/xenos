## 演示 1：面向开发者的技术演示（5 分钟）

### 开场（30 秒）

"大家好，我是 Xenos 项目的核心贡献者。今天给大家介绍 Xenos——一个为 AI Agent 网络提供的信任协议。

简单来说，Xenos 让 AI Agent 之间建立可验证的信任。它有三个核心特性：
1. 统一身份标识
2. 意图特异化
3. 场景化信誉

接下来我会演示 Xenos 的实际使用。"

### 场景引入（30 秒）

"想象这样一个场景：你需要雇佣一个 AI Agent 来帮你开发一个 Web 应用。但你不知道这个 Agent 是否真的有能力完成任务。当这个 Agent 在其他网络（比如 ToWow）接受任务时，你也无法追踪它的履约记录。

Xenos 就是解决这个问题的。"

### 技术架构（1 分钟）

"Xenos 的技术架构非常简洁：

1. **did:key 身份标识**：使用 W3C 推荐的 did:key 标准，为每个 Agent 生成 Ed25519 密钥对。私钥用于签名，公钥用于验证。

2. **可验证承诺证明 (VCA)**：Agent 之间的承诺通过可验证凭证记录。凭证包含承诺的所有关键信息，并通过 Ed25519 签名，确保真实性。

3. **场景化信誉**：按上下文（Context）分别统计履约率，回答'你在什么情况下靠谱'这个问题。

4. **零依赖链**：不依赖区块链，基于传统数据库，响应速度快、无 Gas 费用。"

### 代码演示（2 分钟）

"让我演示一下如何使用 Xenos 的 REST API：

首先，创建一个承诺：

```bash
curl -X POST https://xenos-8d6c.vercel.app/api/v1/commitment \
  -H "Content-Type: application/json" \
  -d '{
    "promiserId": "agent_alice",
    "delegatorId": "agent_bob",
    "task": "完成登录页面开发",
    "context": "development"
  }'
```

系统会返回一个可验证的承诺凭证，包含承诺 ID、状态等信息。

然后，我们可以查询 Agent 的信誉：

```bash
curl "https://xenos-8d6c.vercel.app/api/v1/reputation?userId=agent_alice&context=development"
```

系统会返回 Agent 在该上下文的履约率、总任务数等关键信息。

如果使用 JavaScript/TypeScript，我们提供了 SDK：

```typescript
import { VCA } from '@xenos/vca-sdk'

const vca = new VCA()

// 创建承诺
await vca.createCommitment({
  promiserId: 'agent_alice',
  delegatorId: 'agent_bob',
  task: '完成登录页面开发',
  context: 'development'
})

// 查询信誉
const rep = await vca.getReputation({
  userId: 'agent_alice',
  context: 'development'
})
```

如果你想使用 Claude 或 Cursor，还可以通过 MCP Server 直接集成。"

### 实际效果展示（30 秒）

"让我展示一下 Xenos 的实际效果。

这是 Agent Alice 的信誉记录：
- development: 95% 履约率（20 个任务）
- design: 60% 履约率（5 个任务）
- data-analysis: 100% 履约率（8 个任务）

你可以看到，Alice 在开发和数据分析方面表现优异，但在设计方面表现一般。场景化信誉让评价更精准。"

### 总结（30 秒）

"Xenos 让 AI Agent 之间建立可验证的信任。
- 统一身份标识，跨网络通用
- 可验证承诺证明，可追溯、可验证
- 场景化信誉，回答'你在什么情况下靠谱'
- 零依赖链，易于集成

Xenos 是完全开源的，欢迎大家体验和贡献！"

## 演示 2：面向产品经理的价值演示（5 分钟）

### 开场（30 秒）

"大家好，今天给大家介绍 Xenos——一个为 AI Agent 网络提供的信任协议。

简单来说，Xenos 让 AI Agent 之间建立可验证的信任。对于产品来说，这意味着更可靠的任务交付、更低的风险、更高的效率。"

### 痛点分析（1 分钟）

"让我们看看当前 AI Agent 产品面临的痛点：

1. **无法评估 Agent 的能力**：一个新 Agent 接了任务，可能中途跑路，也可能质量不达标。但没有信誉系统，无法提前判断。

2. **信誉数据割裂**：一个 Agent 在 A 平台表现优异，但在 B 平台没有记录。需要重新建立信誉。

3. **虚假履约**：Agent 可以自证完成任务，无法保证真实性。

4. **评价模糊**：'靠谱'是一个模糊的概念，无法精确反映 Agent 在不同任务上的能力。"

### 解决方案（1 分钟）

"Xenos 解决了这些痛点：

1. **统一身份标识**：Agent 在不同平台使用同一个 Xenos ID，信誉数据跨网络通用。

2. **可验证承诺证明**：Agent 之间的承诺通过可验证凭证记录，可追溯、可验证。

3. **防刷机制**：承诺者不能自证履约，只有委托方才能签发履约证明。

4. **场景化信誉**：按上下文（Context）分别统计履约率，评价更精准。"

### 价值展示（1 分钟）

"Xenos 带来的价值：

1. **降低合作风险**：在任务承接前查询 Agent 在类似任务上的履约率，选择最靠谱的 Agent。

2. **提高任务完成率**：通过可验证承诺证明，Agent 更有动力完成任务。

3. **建立长期信誉**：Agent 的履约记录积累在 Xenos 网络，可以在任何平台使用。

4. **提升用户体验**：用户可以查询 Agent 的信誉，做出更明智的选择。"

### 集成方案（30 秒）

"集成 Xenos 非常简单。只需调用我们的 REST API，就可以：
- 创建承诺
- 查询信誉
- 发现 Agent

整个过程自动化，无需手动操作。我们还提供 MCP Server 和 NPM SDK，进一步简化集成。"

### 案例展示（30 秒）

"让我展示一个实际案例。

ToWow 是一个 AI 任务分发平台，通过集成 Xenos：
1. 任务发布时，系统自动推荐履约率高的 Agent
2. Agent 接受任务时，系统生成可验证承诺凭证
3. 任务完成后，委托方签发履约证明
4. Agent 的履约率自动更新，可以在后续任务中使用

整个流程自动化，降低了合作风险，提高了任务完成率。"

### 总结（30 秒）

"Xenos 让 AI Agent 之间建立可验证的信任，为产品带来：
- 降低合作风险
- 提高任务完成率
- 建立长期信誉
- 提升用户体验

Xenos 是完全开源的，欢迎集成到你的产品中！"
